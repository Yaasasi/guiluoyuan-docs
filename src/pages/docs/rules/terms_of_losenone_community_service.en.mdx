# Terms of Losenone Community Service


## 概述

归落原（“Ourmoe”、“我们” 或 “我们的”）致力于融合所有热爱生活、正直向上与友好善良的人群，努力与所有社区成员们一起构建欢乐且让人留恋的兴趣社群，构建温暖、和谐、友爱、美好、多元的社区，打造主题鲜明、内容真实的社区，鼓励有爱、有趣、有用的内容在社区生长。感谢您参与其中，与我们共同建设！

如果您对《归落原服务条款 / Minecraft 》（以下简称 “本条款” ）现有的条款有任何的疑问（包括实施中的困难）和建议，请通过邮件 help@service.our.moe 与我们取得联系。

## B类 - 社交环境

在本类中，我们实行金字塔阶级处罚措施（共五级），高阶级包含次一级的处罚措施，以此类推：\
   • 第一级：法律手段；\
   • 第二级：永久封禁（社区公开）；\
   • 第三级：限时禁言（24 小时、7 个自然日、1 个自然月）；\
   • 第四级：严重警告（社区通报）；\
   • 第五级：消除影响（撤回或删除）。\
单一社区成员每遭到一次处罚，就会提高一次处罚等级。但是，处罚等级的提高不是绝对线性提高的，我们会根据实际情况的严重程度动态变更，同时我们也会注明其是否以何种形式提高（“0” 代表此前并无处罚等级，即初次受到处罚）：\
   • 线性提高（按照处罚阶级次序提高处罚等级）；\
   • 动态提高（按照实际情况动态提高处罚等级）；\
   • 固定阶级（固定的处罚阶级）。

1. 您在社区约束范围内参与社交活动时必须遵守《[公共社区交流公约](../treaty/community_communication_covenant)》、中华人民共和国的法律法规以及您所处国家的法律法规。

2. 高质量昵称与头像，传递你有趣的灵魂。符合社区及个人特色的昵称和头像可以帮助大家展示自己的风采，快速融入新群体。您对外展示的个人名片不得违反[《公共社区交流公约》第四项 1、2、3、4 款之规定](https://www.our.moe/docs/treaty/community_communication_covenant#%E7%AC%AC%E5%9B%9B%E9%A1%B9-%E4%B8%AA%E4%BA%BA%E5%90%8D%E7%89%87)。\
   • 处罚等级提高方式：线性提高（0 -> 5 -> 4 -> 3 -> 2 -> 1）；\
   • 处罚执行管理员：社区秩序管理员。

3. 在社区约束范围内，不得违反[《公共社区交流公约》第五项 1、2、3、4、5、6 款之规定](https://www.our.moe/docs/treaty/community_communication_covenant#%E7%AC%AC%E4%BA%94%E9%A1%B9-%E4%BA%A4%E6%B5%81%E8%A1%8C%E4%B8%BA)。\
   • 处罚等级提高方式：动态提高（0 -> 3 -> 2 -> 1）；\
   • 处罚执行管理员：社区秩序管理员。

4. 在社区约束范围内，不得违反[《公共社区交流公约》第五项 7、8、9、13、14、15、16、17、18、19、20、21 款之规定](https://www.our.moe/docs/treaty/community_communication_covenant#%E7%AC%AC%E4%BA%94%E9%A1%B9-%E4%BA%A4%E6%B5%81%E8%A1%8C%E4%B8%BA)。\
   • 处罚等级提高方式：线性提高（0 -> 5 -> 4 -> 3 -> 2 -> 1）；\
   • 处罚执行管理员：社区秩序管理员。

5. 在社区约束范围内，不得违反[《公共社区交流公约》第五项 10、11、12 款之规定](https://www.our.moe/docs/treaty/community_communication_covenant#%E7%AC%AC%E4%BA%94%E9%A1%B9-%E4%BA%A4%E6%B5%81%E8%A1%8C%E4%B8%BA)。\
   • 处罚等级提高方式：动态提高（0 -> 5 -> 4 -> 3 -> 2）；\

6. 在社区约束范围内，不得引导其他社区成员加入任何与本社区无关的社群，同时您也不应该在与本社区相同领域的社群宣传本社区。\
   • 处罚等级提高方式：动态提高（0 -> 5 -> 3 -> 2）；\

#### 1. 三方模组

1.1 对于许多玩家来说，Minecraft 模组是影响 Minecraft 游戏体验的一个重要成分，其功能从小的性能和视觉提升到大的改变游戏玩法。虽然 Minecraft 模组有着如此强大的功能与额外内容，但是并非所有的模组都被允许在归落原社区服务器中使用。本规定明确说明并强调了哪些种类的模组允许在归落原社区服务器中使用，哪些种类的模组不允许在归落原社区服务器中使用。
注意，您一旦在归落原社区服务器中使用 Minecraft 模组，则您将承担使用 Minecraft 模组所带来的风险，请您确保您在归落原社区服务器中使用 Minecraft 模组前已仔细阅读、充分理解并毫无保留地接受本规定。

1.2 本规定无法涵盖所有 Minecraft 模组，也无法保证任一模组是符合归落原社区规则的。如果一个 Minecraft 模组不属于下述任一模组类型，那么它默认是不允许在归落原社区服务器中使用的。

1.3 您安装的所有 Minecraft 模组都应当从该模组官方提供的途径获取，如开发者个人网站、GitHub、CurseForge、Modrinth。从非官方渠道获取的 Minecarft 模组可能不会按照模组官方介绍工作，可能包含漏洞或未标明的功能（这可能会导致您在归落原开放的服务器中受到惩罚），甚至包含恶意内容（如病毒、间谍软件等）。

1.4 若我们发现有社区成员使用了本规定明确禁止使用的模组类型，则该社区成员将会被归落原、所属服务器永久封禁，并取消其在归落原中的一切权限，且录入归落原的内部黑名单。

1.5 我们可能由于监管、保护与游戏玩法的原因，禁止使用部分 Minecraft 模组加载器（Minecraft 模组加载器包括但不限于 Forge、Fabric、LiteLoader、Rift、Quilt 与 NeoForge）、Minecraft 第一方客户端 与 Minecraft 第三方客户端（Minecraft 第三方客户端包括但不限于 Badlion Client、Lunar Client）的玩家登录归落原开放的服务器并在归落原开放的服务器中游玩。

1.6 允许使用的模组类型

    - 1.6.1 **优化类模组**\
用于提升 Minecraft 客户端的性能表现的模组，但是这并不包括对 Minecarft 本身做出改变的模组。\
例如 OptiFine（高清修复）、Lithium（锂）、Sodium（钠）、Phosphor（磷）、VulkanMod 等。

    - 1.6.2 **美化类模组**\
用于改变 Minecraft 的视觉与听觉体验的模组，但是这并不包括修改了 Minecraft 游戏玩法的模组。\
例如 ItemPhysic（物品物理掉落）、Shoulder Surfing（越肩视角）、Dynamic Surroundings（动态环境/动态环绕）等。

    - 1.6.3 **信息界面类模组**\
为客户端添加可视化信息界面的模组（如合成表、血量、地图、耐久与药水效果等），但是这并不包括提供了正常玩家无法获得的信息的模组。\
例如 Just Enough Items（JEI 物品管理器）、Journey Map（旅行地图）、AppleSkin（苹果皮）、Durability Viewer（耐久信息显示）、Hwyla（高亮显示）等。

    - 1.6.4 **便利类模组**\
提供便利功能的模组（如容器整理、快捷手势、高清截图等），但是这并不包括为玩家提供不公平优势的模组。\
例如 Inventory Tweaks（一键整理）、Mouse Tweaks（鼠标手势）、Replay（录像回放）等。

5.2 一旦社区成员遭遇盗窃事件，受害人可要求我们进行调查，
